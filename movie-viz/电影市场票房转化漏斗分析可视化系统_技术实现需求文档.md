## 二、实验内容

### 2.1 自拟主题：电影市场票房转化分析

**选题背景**：电影市场是一个典型的"漏斗型"业务场景——每年全球上映数万部电影，但能够获得足够关注、票房达标、实现盈利、获得高口碑的电影数量逐级递减。通过漏斗图可视化这一转化过程，可以直观揭示电影市场的竞争格局和成功要素。

**问题导向**：
- 不同类型电影（动作、喜剧、科幻等）的票房转化效率有何差异？
- 哪些环节是电影票房转化的主要瓶颈？
- 如何通过数据驱动的方式为电影投资和制作提供建议？

**价值性**：为电影制作方、投资方、发行方提供数据参考，辅助决策。

### 2.2 数据采集与预处理

#### 2.2.1 数据源说明

**数据来源**：IMDB（互联网电影数据库）公开数据集

**数据链接**：https://datasets.imdbws.com/

**数据文件**：

| 文件名 | 内容说明 | 主要字段 |
|--------|----------|----------|
| `title.basics.tsv.gz` | 电影基础信息 | tconst, titleType, primaryTitle, originalTitle, startYear, runtimeMinutes, genres |
| `title.ratings.tsv.gz` | 电影评分数据 | tconst, averageRating, numVotes |

**数据获取方式**：直接从上述链接下载 TSV.GZ 压缩文件，解压后使用。

#### 2.2.2 数据预处理流程

```
原始数据（TSV）
    ↓
1. 解析：使用 Papa Parse 库解析 TSV 格式
    ↓
2. 筛选：仅保留 titleType = "movie" 的记录
    ↓
3. 清洗：去除 genres 为空、runtimeMinutes 为 "\N" 的记录
    ↓
4. 关联：将 basics 与 ratings 通过 tconst 进行关联
    ↓
5. 聚合：按 genres 分组统计各类型电影数量、平均评分、平均投票数
    ↓
6. 转化：构建漏斗数据模型（上映→高关注→高评分→高口碑）
```

### 2.3 图形选择：漏斗图

#### 2.3.1 为何选择漏斗图

| 选择理由 | 详细说明 |
|----------|----------|
| **天然适配转化场景** | 电影市场存在清晰的"上映→关注→好评→经典"递减链路，漏斗图能直观展示各层级的数量递减关系 |
| **强调层级对比** | 漏斗图通过面积差异突出各层级的数量变化，便于识别转化瓶颈 |
| **支持交互分析** | ECharts 漏斗图支持 tooltip、click 等交互，可点击查看各层级详细数据 |
| **视觉冲击力强** | 漏斗形状本身具有强烈的视觉引导效果，能快速传达"逐级筛选"的核心信息 |

#### 2.3.2 为何不选其他图形

| 图形类型 | 不选择的原因 |
|----------|--------------|
| **雷达图** | 适合多维度指标对比（如电影的评分、时长、年份），但不适合展示数量递减的转化流程 |
| **仪表盘** | 适合展示单一指标的完成度（如票房目标达成率），但无法展示多层级的转化关系 |
| **旭日图** | 适合展示层级包含关系（如洲→国家→城市），但电影数据更适合线性漏斗而非嵌套圆环 |
| **关系图** | 适合展示实体间的关联网络（如演员合作关系），不适合数量递减分析 |
| **复合图** | 可以组合使用，但核心转化关系仍以漏斗图为主，复合图为辅 |

## 四、数据结构设计

### 4.1 原始数据结构（IMDB TSV）

#### title.basics.tsv 字段说明

```
tconst          - 电影唯一标识（如 tt0000001）
titleType       - 类型（movie, short, tvSeries, etc.）
primaryTitle    - 主要标题
originalTitle   - 原始标题
isAdult         - 是否成人内容（0/1）
startYear       - 上映年份
endYear         - 结束年份（电影为 \N）
runtimeMinutes  - 时长（分钟）
genres          - 类型标签（逗号分隔，如 Action,Comedy）
```

#### title.ratings.tsv 字段说明

```
tconst          - 电影唯一标识（关联字段）
averageRating   - 平均评分（1-10）
numVotes        - 投票数量
```

### 4.2 处理后数据结构

#### 电影对象（前端处理后）

```javascript
class Movie {
    constructor(basics, ratings) {
        this.id = basics.tconst;                // 电影唯一ID
        this.title = basics.primaryTitle;       // 电影名称
        this.year = parseInt(basics.startYear); // 上映年份
        this.runtime = parseInt(basics.runtimeMinutes); // 时长
        this.genres = basics.genres.split(","); // 类型数组
        this.rating = ratings ? parseFloat(ratings.averageRating) : 0; // 评分
        this.votes = ratings ? parseInt(ratings.numVotes) : 0;         // 投票数
    }
}
```

#### 漏斗层级数据结构

```javascript
class FunnelLevel {
    constructor(name, count, percent) {
        this.name = name;       // 层级名称
        this.value = count;     // 数量
        this.percent = percent; // 占比（相对于第一层）
    }
}
```

#### 5.2.3 漏斗数据计算

```javascript
// 计算漏斗数据
function calculateFunnelData(movies) {
    const totalMovies = movies.length;
    
    const withRatings = movies.filter(m => m.votes >= 100).length;
    const mediumRating = movies.filter(m => m.votes >= 100 && m.rating >= 6.0).length;
    const highRating = movies.filter(m => m.votes >= 100 && m.rating >= 7.5).length;
    const classics = movies.filter(m => m.votes >= 10000 && m.rating >= 8.5).length;
    
    return [
        { 
            name: '上映电影总数', 
            value: totalMovies, 
            percent: '100%',
            conversionRate: '-'
        },
        { 
            name: '获得评分电影', 
            value: withRatings, 
            percent: (withRatings / totalMovies * 100).toFixed(2) + '%',
            conversionRate: '100%'
        },
        { 
            name: '中等口碑电影', 
            value: mediumRating, 
            percent: (mediumRating / totalMovies * 100).toFixed(2) + '%',
            conversionRate: (mediumRating / withRatings * 100).toFixed(2) + '%'
        },
        { 
            name: '高口碑电影', 
            value: highRating, 
            percent: (highRating / totalMovies * 100).toFixed(2) + '%',
            conversionRate: (highRating / mediumRating * 100).toFixed(2) + '%'
        },
        { 
            name: '经典佳作', 
            value: classics, 
            percent: (classics / totalMovies * 100).toFixed(2) + '%',
            conversionRate: (classics / highRating * 100).toFixed(2) + '%'
        }
    ];
}
```

## 六、可视化分析与解读

### 6.1 核心信息描述

通过漏斗图可视化，预期呈现以下核心信息：

1. **转化率递减**：从上映电影总数到经典佳作，数量逐级递减，符合"漏斗型"市场特征
2. **关键瓶颈**：识别转化率最低的环节（如"获得评分"到"中等口碑"可能是主要瓶颈）
3. **类型差异**：不同电影类型（动作、喜剧、科幻等）的漏斗形态可能存在显著差异

### 6.2 可行性分析

| 维度 | 分析 |
|------|------|
| **数据可行性** | IMDB 数据集覆盖全球数百万部电影，数据量充足，字段完整 |
| **技术可行性** | ECharts 漏斗图功能成熟，Papa Parse 可高效解析 TSV 格式 |
| **展示可行性** | 漏斗图直观展示转化关系，支持交互筛选，用户体验良好 |

### 6.3 潜在问题

1. **数据偏差**：IMDB 数据以欧美电影为主，可能存在地域偏差
2. **评分标准**：不同年代的评分标准可能存在差异
3. **票房缺失**：IMDB 数据集不包含票房数据，需用评分和投票数作为代理指标
4. **类型标签**：部分电影可能有多个类型标签，分类可能存在重叠

### 6.4 业务建议

基于漏斗分析结果，提出以下建议：

1. **投资方向**：关注转化率较高的电影类型，如剧情片、纪录片等
2. **质量把控**：确保电影达到"中等口碑"门槛（评分 >= 6.0），这是进入良性循环的关键
3. **营销策略**：提高电影的曝光度和投票数，增加进入漏斗上层的机会
4. **长期价值**：追求"经典佳作"标准，实现长期品牌价值
