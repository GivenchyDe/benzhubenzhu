一．实验目的
1掌握智能合约核心业务逻辑设计学会设计并实现单车管理、租赁计费、押金冻结 / 退还、费用结算等完整租赁流程的合约规则，理解链上状态变更与权限控制。
2掌握 Bike3 前端与链上交互开发能够完成钱包管理、账户创建 / 导入 / 切换、链下数据持久化、合约调用、事件监听、数据展示等全流程 Bike3 开发。
3掌握以太坊钱包与交易基础理解以太坊地址、私钥、助记词、余额管理、模拟交易与上链日志机制，熟悉 ethers.js 的使用。
4实现完整业务闭环完成从运营方添加单车 → 用户租车扣费 → 计时计费 → 还车退款 → 故障上报与修复 → 数据统计的全流程实验，验证系统可用性。
5理解区块链实验的本地模拟方法掌握用 localStorage 模拟链上存储、用前端模拟合约执行、脱离真实公链的安全实验方式
二．实验内容
1. 智能合约
设计并实现共享单车租赁核心合约逻辑，定义单车、订单、交易数据结构。实现运营方添加单车、用户租车押金冻结、按分钟计费、还车扣费退款、故障上报与修复功能。通过权限控制区分用户与管理员，记录链上交易与事件，保障押金安全、计费透明、交易可追溯，完成去中心化租赁全流程合约规则。
2. 前端 Web 技术
使用 HTML、CSS、JavaScript 搭建响应式 Bike3 界面，集成 ethers.js 实现钱包创建、导入、切换与模拟充值。开发首页统计、单车列表、订单管理、运营后台、链上日志模块。实现租还车、故障上报、数据刷新交互，用 localStorage 持久化数据，搭配弹窗与提示优化体验，完成前端与模拟合约的完整交互。
